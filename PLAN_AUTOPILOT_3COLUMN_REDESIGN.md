# 오토파일럿 페이지 3열 레이아웃 재디자인 기획안

## 📊 현재 문제점 분석

### 현재 레이아웃 (1열 세로 스크롤)
- ✅ 장점: 단순하고 직관적
- ❌ 단점:
  - 좌우 공간 낭비 (max-w-5xl로 제한)
  - 세로 스크롤이 길어짐 (5개 섹션 + 로그)
  - 설정과 결과를 동시에 볼 수 없음
  - 화면 활용도 낮음 (특히 와이드 모니터)

---

## 🎯 재디자인 목표

1. **스크롤 최소화** - 대부분의 컨텐츠를 한 화면에 표시
2. **공간 효율 극대화** - 좌우 공간 활용
3. **워크플로우 가시성** - 설정 → 진행 → 결과를 한눈에
4. **반응형 유지** - 작은 화면에서도 사용 가능

## 📐 상세 레이아웃 설계

### LEFT COLUMN (30% - 설정 입력)
**컴팩트한 폼 요소들**

1. **주제 설정** (1️⃣)
   - 키워드 입력 필드
   - 최소 높이로 압축

2. **나레이션 & 오디오** (4️⃣)
   - 대본 스타일 (Select)
   - 영상 길이 (Number Input)
   - 음성 제공자 (Select)
   - 음성 선택 (Select)
   - 2열 그리드로 압축

3. **자막 설정** (5️⃣)
   - 현재 설정 미리보기 (축소)
   - 접을 수 있는 아코디언 형태

### CENTER COLUMN (40% - 스타일 선택)
**비주얼 중심의 선택 영역**

1. **이미지 스타일** (2️⃣)
   - 그리드 레이아웃 (4-5열)
   - 스크롤 가능한 영역
   - 선택된 스타일 하이라이트

2. **썸네일 스타일** (3️⃣)
   - 그리드 레이아웃 (4-5열)
   - 스크롤 가능한 영역
   - 선택된 스타일 하이라이트

3. **비디오 장면 수** (슬라이더)
   - 하단에 컴팩트하게 배치

### RIGHT COLUMN (30% - 실행 & 모니터링)
**실행 제어 및 피드백**

1. **시작 버튼** (상단 고정)
   - 큰 CTA 버튼
   - 현재 상태 표시

2. **진행 상태** (중간)
   - 단계별 진행률
   - 현재 작업 표시
   - 프로그레스 바

3. **실시간 로그** (하단)
   - 스크롤 가능한 콘솔
   - 자동 스크롤
   - 로그 지우기 버튼

## 🎯 핵심 개선 사항

### 1. 스크롤 제거
- **Before**: 5개 섹션이 세로로 나열 → 긴 스크롤
- **After**: 3열로 분산 → 대부분 한 화면에 표시

### 2. 공간 활용
- **Before**: max-w-5xl로 중앙 제한 → 좌우 공백
- **After**: 전체 너비 활용 → 3열 분할

### 3. 워크플로우 가시성
- **Before**: 설정 → 스크롤 → 로그 확인
- **After**: 설정 | 스타일 | 실행/로그 동시 표시

### 4. 우선순위 배치
- **왼쪽**: 텍스트 입력 (덜 중요)
- **중앙**: 비주얼 선택 (중요)
- **오른쪽**: 실행 & 피드백 (가장 중요)

## 💡 추가 제안

### 1. 접을 수 있는 섹션
- 자막 설정 등 덜 중요한 섹션은 아코디언으로
- 필요할 때만 펼쳐서 공간 절약

### 2. 미니맵 네비게이션
- 우측 상단에 작은 미니맵
- 현재 진행 단계 표시

### 3. 키보드 단축키
- `Ctrl + Enter`: 시작
- `Ctrl + L`: 로그 지우기
- `Tab`: 섹션 간 이동

### 4. 다크모드 최적화
- 3열 레이아웃에서 각 컬럼 구분 명확히
- 미묘한 배경색 차이로 영역 구분
