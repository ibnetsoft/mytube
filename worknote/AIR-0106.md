## Goal
- Audit whether the real Longform production pipeline can carry a user from topic selection through externally generated image/video asset import without losing scene order.
- Establish the correct product boundary between AIR Studio AI and external generation services.

## Context
- AIR Studio AI owns planning, script generation, scene splitting, image prompts, and video prompts.
- Users create images, upscale images, and generate clips through external services.
- This Task ID was also used on an unmerged provider-prompt branch. This branch records the separately requested production-pipeline audit so the duplicate ID remains visible rather than hidden.

## Tests / Validation
- Read the Longform product and task-control documents.
- Traced `/image-crop` and `POST /api/settings/crop-grid`.
- Traced image prompt persistence and `scene_number` ordering in `database.py`.
- Traced single-scene image/video upload endpoints.
- Traced `/api/image/bulk-match` and `services/video_matcher.py`.
- Verified behavior by static code inspection; no production data was mutated.

## Result
- Documented the end-to-end production workflow and ownership boundary.
- Classified each stage as implemented, partially implemented, external/manual, or missing.
- Confirmed that scene rows load in number order after assignment.
- Identified import safety as the first production blocker:
  - no deterministic filename-first matching
  - no duplicate or occupied-slot protection
  - no complete unmatched or missing-scene report
  - silent last-write-wins behavior
- Moved these fixes into the AIR-0107 handoff.

## Known Issues
- 2x2 crop results must be downloaded and manually re-imported.
- Scene identity can change when prompt rows are deleted and renumbered.
- Bulk matching depends on Gemini availability.
- Legacy direct generation controls remain visible in Longform UI.
- External upscaling provenance is not tracked.

## Next Action
- AIR-0107: implement deterministic, validated, non-destructive bulk scene asset import before expanding the pipeline.
