# AIR-0104

Date: 2026-06-27

## Goal
Validate the real `Longform Mode` `/script-plan` flow, block invalid cross-mode entry, and tighten `project_id` access around the plan screen and its supporting APIs.

## Context
- `AIR-0103` separated `/script-plan` and `/music-plan` by selected project mode.
- The next risk was that project pages and APIs still trusted raw `project_id` values too broadly.
- The `script_plan` template still carried nursery/song-specific UI fragments that should not appear in normal `Longform Mode`.

## Files Changed
- `app/project_access.py`
- `app/routers/pages.py`
- `app/routers/projects.py`
- `app/routers/gemini.py`
- `main.py`
- `static/js/api.js`
- `static/js/api_v2.js`
- `templates/pages/script_plan.html`
- `tests/test_longform_script_plan_access.py`
- `tests/test_project_music_integration.py`

## Tests / Validation
- `venv\Scripts\python.exe -m pytest tests\test_longform_script_plan_access.py -q`
- `venv\Scripts\python.exe -m pytest tests\test_project_music_integration.py -q`

Validated:
- Longform project -> `/script-plan` returns `200`
- Music project -> `/script-plan` redirects to `/music-plan`
- Longform project -> `/music-plan` redirects back to `/script-plan`
- Standard membership -> `/music-plan` redirects to `/projects`
- Foreign `project_id` access is blocked for `/api/projects/{id}` and `/api/projects/{id}/full`
- Longform `/script-plan` HTML no longer renders the nursery idea panel or nursery style option

## Result
- Added shared project-access enforcement and reused it in page routing, project APIs, settings/metadata/full-data APIs, and structure-generation endpoints.
- Improved frontend project API wrappers so unauthorized or failed project-context loads surface as thrown errors instead of silently returning malformed JSON payloads.
- Removed the visible nursery-rhyme option/panel from normal longform script planning and kept payout formatting aligned with `$` output.

## Known Issues
- Browser-level manual verification was not run in this task; validation used `TestClient` and rendered HTML assertions.
- The rest of the longform page family (`image-gen`, `audio-gen`, `render`, `title-desc`, upload/export) still needs the same project-aware access and mode cleanup.
- `templates/pages/script_plan.html` still contains some legacy nursery/music helper JS and mojibake comments even though the visible UI is now hidden from longform users.

## Next Action
- `AIR-0105`: continue project-aware route/access cleanup across the remaining longform worker pages and standardize the final export/publish contract for standard members.
